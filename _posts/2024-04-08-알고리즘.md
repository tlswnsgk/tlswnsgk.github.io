1.자료구조 개념
-자료구조는 그릇(자료)을 효율적으로 관리하는 방법

2.알고리즘은?
-목적지까지 최적의 이동 경로를 찾는 방법

3.자료구조의 개념
컴퓨터 프로그래밍 언어에서 효율적인 자료(데이터)의 형태

```

- 단순 자료구조
 정수, 실수, 문자, 문자열

- 선형 자료구조
리스트, 스택, 큐

- 비선형 자료구조
 트리, 그래프 

- 파일 자료구조
 순차 파일, 색인 파일, 직접 파일.
 
```

2.자료구조 종류

```

01 단순 자료구조 
    프로그래밍 언어의 데이터 형식에 해당하는 정수,실수,문자,문자열 등 
    - 정수 : int 또는 integer 형태로 소수점이 없음
    ex 0, 100, 1234, -27 등 

    - 실수 : 소수점이 있는 형태로 float 등으로 표현
    ex 0.1 , 3.14, 1.234567 등

    - 문자 : 한 글자를 의미하며, char로 표현, 주로 작은따옴표('')로 묶어줌
    ex 'A', '채', '3', 등

    - 문자열 : 글자 여러 개를 연결한 것으로, string으로 주로 큰따옴표("")로 표현 
    ex "안녕" 등

02 선헝 자료구조
    데이터를 한 줄로 순차적으로 표현한 형태 : 선형리스트,연결시트르,스택,큐 등
    A-B-C-D-E

03 비선형 자료구조
    하나의 데이터 뒤에 여러 개가 이어지는 형태: 트리와 그래프 등


04 파일 자료구조
 (2)파일 내용이 디스크에 저장되는 방식에 따라 순차 파일과 직접 파일로 구분

파일 내용을 임의의 물리적 위치에 기록하는 방식으로 직접 접근 방식(Direct Access Method)

 (3) 색인 순차 파일
    순차파일과 직접 파일이 결합된 형태
```

3.알고리즘 개념 <br>

```

    알고리즘이란
    01. 알고리즘은 문제 해결 과정을 묘사하는 것
        -문제 해결 절차를 체계적으로 기술한 것
        -문제의 요구 조건
        1. 입력과 출력으로 명시할 수 있다
        2. 알고리즘은 입력으로부터 출력을 만드는 과정을 기술
    02. 입출력의 예
        1. 문제 : 100명의 학생의 시험점수와 최대값을 찾으라
        2. 입력 : 100명의 학생들의 시험점수
        3. 출력 : 위 100개의 시험점수들 중 최대값
    03. 알고리즘은 자료구조의 확장
        선행과목 : 프로그래밍, 자료구조
        자료구조
            1. 건축의 건축자재나 모듈 같은 것
            2. 자동차 제작이 부품이나 모듈 같은 것
    04. 자료구조
        컴퓨터분야에서효율적으로접근하고수정할수있도록자료를구성·관리·저장하는것
    05. 알고리즘
        컴퓨터분야나수학등관련분야에서어떤문제를해결하기위해정해진일련의단계적인절차,방법
    06. 알고리즘과자료구조
        자료구조와알고리즘과프로그램의관계
    07. 알고리즘개요
        1 알고리즘의정의
            ① 문제를해결하거나함수를계산하기위해좇아야할모호함이없는
              간단한명령들로구성된일련의순차적단계
            ② 특정한작업을수행하기위한유한명령어들로구성되며
              컴퓨터의수행에적합한문제해결을위한방법
        2 알고리즘의목표
            ① 단순히원하는결과를얻을수있는알고리즘이아닌, 
            처리시간이나기억장소사용측면에서효율적인알고리즘을개발하는것이주요목표
            ② 효율성: 효율적인해결방법들을발견하여활용함
            ③ 추상화: 복잡한문제도효율적인알고리즘으로해결할수있는단순한문제들로분류가능
            ④ 재사용성: 단순한문제를효율적해결가능한알고리즘은다른많은문제해결에도활용
    10.  알고리즘개요
        4 알고리즘의조건
        ① 입출력, 명확성, 유한성, 효과성을만족하는알고리즘이존재하면그문제가해결가능하지만
        알고리즘은궁극적으로컴퓨터에서수행하는것을목표로하므로
        실용성이있어야하고효율성도필요
        ② 단순히원하는결과를얻을수있는알고리즘이아닌, 
        처리시간이나기억장소사용측면에서효율적인알고리즘을개발하는것이주요목표

         알고리즘의생성절차: 알고리즘설계→ 표현→ 정확성검증→ 효율성분석

```

```

4. 알고리즘 표현법
   1.   일반언어표현
       V 일반적인자연어를사용해서설명하듯이알고리즘을표현
       V 일반사람이이해하기쉽게표현할수있으나, 
       최종적으로코드로변경하는데는한계가있음
       V 어떤알고리즘을사용해야할지아이디어가떠오르지않는시점에서, 
       생각범위를넓히는단계정도에사용하면무난

   2.   순서도를이용한표현
       V 여러종류의상자와상자를 이어주는화살표를이용해서 명령순서를표현.

       V 간단한알고리즘은 쉽게표현할수있지만, 복잡한알고리즘은 표현하기어려운경우가많음

   3.   의사코드를이용한표현
       V 프로그래밍언어보다는 좀더인간의언어에가까운형태
       V 프로그램코드와일반언어의중간형태
       V 프로그램코드를직접코딩하는것보다 알고리즘을좀더명확하게정립하는데 도움이되고코드에설명을달지않아도 이해하는데무리없음

   4.   프로그램코드로표현
       V 실제로사용하는프로그래밍언어의코드로바로작성가능

   5.   혼합형태
       V 간단한알고리즘은직접코드로작성
       V 복잡한알고리즘은일반언어, 의사코드, 순서도, 그림등을종합적으로활용해서표현

```
